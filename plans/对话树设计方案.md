# NPC 对话树设计方案

## 一、现状分析

### 1.1 现有对话系统架构

当前 [`DialogueSystem.cs`](Common/Systems/DialogueSystem.cs) 是一个基于 **2按钮 + 5选择** 的扁平对话系统：

```
┌─────────────────────────────────────────────┐
│  GuMasterBase.SetChatButtons()              │
│    ↓                                         │
│  DialogueSystem.GenerateDialogueOptions()    │
│    ├─ button  = "询问" (BUTTON_ASK)          │
│    └─ button2 = "交易" (BUTTON_TRADE)        │
│                                              │
│  GuMasterBase.OnChatButtonClicked()          │
│    ↓                                         │
│  DialogueSystem.OnDialogueChoice(index)      │
│    ├─ 0 = 诚实展示修为 (HandleHonestReveal)  │
│    ├─ 1 = 虚报修为     (HandleDeception)     │
│    ├─ 2 = 拒绝回答     (HandleRefusal)       │
│    ├─ 3 = 威胁         (HandleThreat)        │
│    └─ 4 = 贿赂         (HandleBribe)         │
└─────────────────────────────────────────────┘
```

### 1.2 现有系统的局限性

| 问题 | 描述 |
|------|------|
| **扁平结构** | 只有一层对话选择，无法实现多轮对话树分支 |
| **硬编码选择** | 5个选择是硬编码的，无法根据 NPC 性格/玩家状态动态调整 |
| **无对话节点概念** | 没有对话节点（DialogueNode）的数据结构，难以扩展 |
| **UI 受限** | 依赖原版 2 按钮系统，无法展示多选项对话树 |
| **无条件分支** | 对话选项不能根据条件（信念、声望、修为）动态显示/隐藏 |
| **无对话历史** | 没有记录对话历史，无法实现上下文相关的对话 |

### 1.3 现有可复用资源

- [`BeliefState`](Content/NPCs/GuMasters/IGuMasterAI.cs:35) — 信念系统（RiskThreshold / ConfidenceLevel / EstimatedPower）
- [`GuAttitude`](Common/Systems/GuWorldSystem.cs:106) — 态度枚举（Hostile / Wary / Ignore / Friendly / Fearful / Respectful / Contemptuous）
- [`GuPersonality`](Content/NPCs/GuMasters/IGuMasterAI.cs) — 性格系统（Aggressive / Cautious / Greedy / Proud / Benevolent）
- [`GuWorldPlayer`](Common/Players/GuWorldPlayer.cs) — 声望系统（FactionReputation / InfamyPoints）
- [`QiRealmPlayer`](Common/Players/QiRealmPlayer.cs) — 修为系统（GuLevel）
- [`SearchPlayer`](Common/Players/SearchPlayer.cs) — 搜索技能系统（Speed / Precision / Awareness / Knowledge）

## 二、对话树核心设计

### 2.1 对话节点（DialogueNode）数据结构

```csharp
/// <summary>
/// 对话节点 — 对话树的基本单元
/// </summary>
public class DialogueNode
{
    /// <summary> 节点唯一标识 </summary>
    public string NodeID;
    
    /// <summary> NPC 说的文本（支持格式化占位符） </summary>
    public string NPCText;
    
    /// <summary> 可选的对话选项列表 </summary>
    public List<DialogueOption> Options;
    
    /// <summary> 进入此节点时触发的效果 </summary>
    public List<DialogueEffect> OnEnterEffects;
    
    /// <summary> 离开此节点时触发的效果 </summary>
    public List<DialogueEffect> OnExitEffects;
    
    /// <summary> 是否终止对话（关闭对话界面） </summary>
    public bool EndsDialogue;
    
    /// <summary> 是否触发战斗 </summary>
    public bool TriggersCombat;
    
    /// <summary> 是否打开商店 </summary>
    public string OpensShop;
    
    /// <summary> 节点标签（用于调试/日志） </summary>
    public string Label;
}
```

### 2.2 对话选项（DialogueOption）数据结构

```csharp
/// <summary>
/// 对话选项 — 玩家可以选择的一个回答
/// </summary>
public class DialogueOption
{
    /// <summary> 选项显示文本 </summary>
    public string Text;
    
    /// <summary> 目标节点 ID </summary>
    public string TargetNodeID;
    
    /// <summary> 显示此选项的条件（为空则始终显示） </summary>
    public DialogueCondition Condition;
    
    /// <summary> 选择此选项时触发的效果 </summary>
    public List<DialogueEffect> Effects;
    
    /// <summary> 选项类型（用于 UI 样式区分） </summary>
    public DialogueOptionType OptionType;
    
    /// <summary> 工具提示文本（鼠标悬停时显示） </summary>
    public string Tooltip;
}
```

### 2.3 对话条件（DialogueCondition）系统

```csharp
/// <summary>
/// 对话条件 — 决定选项是否可见/可选
/// </summary>
public abstract class DialogueCondition
{
    /// <summary> 评估条件是否满足 </summary>
    public abstract bool Evaluate(Player player, NPC npc, BeliefState belief);
}

// 具体条件类型：
public class BeliefCondition : DialogueCondition  // 信念条件（RiskThreshold > 0.5 等）
public class ReputationCondition : DialogueCondition  // 声望条件（FactionReputation > 50 等）
public class RealmCondition : DialogueCondition  // 修为条件（GuLevel >= 3 等）
public class QuestCondition : DialogueCondition  // 任务条件（HasQuest("xxx") 等）
public class HasItemCondition : DialogueCondition  // 物品条件（HasItem(ItemID.xxx) 等）
public class AndCondition : DialogueCondition  // 与条件
public class OrCondition : DialogueCondition  // 或条件
public class NotCondition : DialogueCondition  // 非条件
```

### 2.4 对话效果（DialogueEffect）系统

```csharp
/// <summary>
/// 对话效果 — 选择选项后触发的副作用
/// </summary>
public abstract class DialogueEffect
{
    /// <summary> 执行效果 </summary>
    public abstract void Execute(Player player, NPC npc, BeliefState belief);
}

// 具体效果类型：
public class ModifyBeliefEffect : DialogueEffect  // 修改信念（RiskThreshold ±0.1 等）
public class ModifyReputationEffect : DialogueEffect  // 修改声望
public class GiveItemEffect : DialogueEffect  // 给予物品
public class RemoveItemEffect : DialogueEffect  // 移除物品
public class SetFlagEffect : DialogueEffect  // 设置对话标记
public class StartQuestEffect : DialogueEffect  // 触发任务
public class PlaySoundEffect : DialogueEffect  // 播放音效
public class ShowMessageEffect : DialogueEffect  // 显示消息
```

public enum DialogueOptionType

### 3.2 基于代码的对话树定义（复杂逻辑时使用）

对于需要复杂条件判断的对话树，提供 C# 构建器 API：

```csharp
public class GuYuePatrolDialogueTree : DialogueTreeBuilder
{
    public override string TreeID => "GuYuePatrolGuMaster_Dialogue";
    
    public override DialogueNode BuildRoot(Player player, NPC npc)
    {
        var belief = GetBelief(npc, player);
        
        return DialogueNode.Create("greeting")
            .WithText($"{npc.GivenName}看了你一眼：\"有什么事？\"")
            .WithOption("打听消息", "ask_info", DialogueOptionType.Informative)
            .WithOption("交易", "open_shop", DialogueOptionType.Trade,
                condition: new BeliefCondition(b => b.HasTraded))
            .WithOption("展示修为", "reveal_power")
            .WithOption("离开", null, DialogueOptionType.Exit)
            .Build();
    }
}
```

## 六、对话树与信念系统的深度整合

### 6.1 信念感知的对话分支

```
玩家选择"展示修为"
    │
    ├─ EstimatedPower < 0.3（NPC 觉得你弱）
    │   └─ NPC 轻蔑："就这点修为也敢出来走动？"
    │       ├─ 愤怒反击 → 触发战斗
    │       └─ 默默离开 → 回到上级
    │
    ├─ EstimatedPower 0.3~0.7（NPC 觉得你差不多）
    │   └─ NPC 警惕："修为不错，但别想耍花招。"
    │       ├─ 提议切磋 → 友好战斗（不致死）
    │       └─ 交换情报 → 情报节点
    │
    └─ EstimatedPower > 0.7（NPC 觉得你很强）
        └─ NPC 恭敬："前辈大驾光临，有失远迎！"
            ├─ 索要供奉 → 获得物品
            ├─ 发布命令 → 任务节点
            └─ 收为随从 → 随从系统（P2）
```

### 6.2 性格驱动的对话风格

| 性格 | 对话风格 | 特殊选项 |
|------|---------|---------|
| [`Aggressive`](Content/NPCs/GuMasters/IGuMasterAI.cs:225) | 直接、好斗 | 挑衅、决斗 |
| [`Cautious`](Content/NPCs/GuMasters/IGuMasterAI.cs:226) | 谨慎、试探 | 谈判、结盟 |
| [`Greedy`](Content/NPCs/GuMasters/IGuMasterAI.cs:227) | 贪婪、算计 | 贿赂、交易 |
| [`Proud`](Content/NPCs/GuMasters/IGuMasterAI.cs:228) | 高傲、轻蔑 | 挑衅、臣服 |
| [`Benevolent`](Content/NPCs/GuMasters/IGuMasterAI.cs:229) | 仁慈、友善 | 求助、指导 |

### 6.3 态度决定的对话入口

```
玩家与 NPC 对话
    │
    ├─ GuAttitude.Hostile → 无法对话（CanChat = false）
    │
    ├─ GuAttitude.Wary → 对话树根节点为"警惕问候"
    │   └─ 选项受限（大部分情报/交易选项不可用）
    │
    ├─ GuAttitude.Ignore → 对话树根节点为"冷淡问候"
    │   └─ 基础选项可用
    │
    ├─ GuAttitude.Friendly → 对话树根节点为"友好问候"
    │   └─ 大部分选项可用
    │
    ├─ GuAttitude.Respectful → 对话树根节点为"恭敬问候"
    │   └─ 所有选项可用 + 特殊尊敬选项
    │
    └─ GuAttitude.Fearful → 对话树根节点为"恐惧问候"
        └─ 选项偏向威胁/勒索
```

## 七、对话树示例

### 7.1 古月巡逻蛊师完整对话树

```
[greeting] 有什么事？
  ├─ [ask_info] 打听消息
  │   ├─ [info_faction] 附近势力 → 获得情报 → 回到 greeting
  │   ├─ [info_treasure] 宝物下落（声望≥30）→ 获得情报 → 回到 greeting
  │   └─ [back] 算了 → 回到 greeting
  │
  ├─ [trade] 交易（已交易过）
  │   └─ 打开商店 → 结束对话
  │
  ├─ [reveal_power] 展示修为
  │   ├─ 弱 → [mock] 被嘲讽
  │   │   ├─ 愤怒 → 触发战斗
  │   │   └─ 隐忍 → 回到 greeting
  │   ├─ 中 → [neutral] 被认可
  │   │   ├─ 切磋 → 友好战斗
  │   │   └─ 交换情报 → 进入 info 分支
  │   └─ 强 → [respect] 被尊敬
  │       ├─ 索要供奉 → 获得物品
  │       └─ 发布命令 → 任务
  │
  ├─ [probe] 试探底细（情报技能≥30）
  │   ├─ 成功（精度判定）→ 获得信念信息
  │   └─ 失败 → NPC 警惕，RiskThreshold +0.05
  │
  ├─ [threaten] 威胁（自信度 > 0.5）
  │   ├─ 高傲性格 → 触发战斗
  │   └─ 其他性格 → 勒索成功，获得小额物品
  │
  └─ [leave] 离开 → 结束对话
```

## 十、与现有系统的关系

### 10.1 与信念系统的关系

```
BeliefState  ←────────── 对话树读取信念状态决定分支
    ↑
    │
DialogueEffect.ModifyBelief  ────→  修改信念
    │
    ↓
GuAttitudeHelper.CalculateFromBelief  ────→  态度变化
    │
    ↓
下一帧 AI 循环感知态度变化 → NPC 行为改变
```

### 10.2 与搜索系统的关系

```
SearchPlayer (搜索技能)
    │
    ├─ Awareness ≥ 30 → 对话中出现"试探底细"选项
    ├─ Knowledge ≥ 50 → 对话中情报选项更多
    └─ Precision ≥ 40 → 试探成功率更高
```

### 10.3 与声望系统的关系

```
GuWorldPlayer.FactionRelations
    │
    ├─ 声望 ≥ 30 → 解锁情报选项
    ├─ 声望 ≥ 50 → 解锁特殊交易选项
    ├─ 声望 ≥ 80 → 解锁结盟选项
    └─ 声望 < 0  → 对话树根节点变为敌对问候
```
