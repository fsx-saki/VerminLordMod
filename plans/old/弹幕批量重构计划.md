## 弹幕分类与重构方案

### 批次 1：纯直线飞行 + 拖尾（约 25 个，最简单）

这些弹幕的 AI 只有 `trailManager.Update + rotation`，OnSpawn 初始化拖尾，PreDraw 调用 trailManager.Draw。

**模板代码**（每个弹幕约 60 行 → 约 30 行）：

```csharp
class XXXProj : BaseBullet
{
    protected override void RegisterBehaviors()
    {
        Behaviors.Add(new AimBehavior(speed: 0f) { AutoRotate = true });
        Behaviors.Add(new TrailBehavior { AutoDraw = true });
    }

    public override void SetDefaults() { /* 标准配置 */ }

    protected override void OnSpawned(IEntitySource source)
    {
        var trail = Behaviors.Find(b => b is TrailBehavior) as TrailBehavior;
        trail.TrailManager.AddGhostTrail(trailTex, color: ..., ...);
    }
}
```

## 推荐执行顺序

```
阶段 1：实现缺失的行为
  ├── HomingBehavior（CycloneProj 的追踪 AI）
  ├── GravityBehavior（StarDartProj 的重力）
  ├── BounceBehavior（StarDartProj 的碰撞反弹）
  └── WaveBehavior（ShaZhaoJiuYueZhan 的波浪）

阶段 2：批次 1 — 25 个纯直线飞行弹幕（每个 2-3 分钟）
  └── 模板化重构，批量替换

阶段 3：批次 2 — 5 个发光弹幕（每个 5 分钟）
  ├── MeteorProj（+RotateBehavior）
  ├── CycloneProj（+HomingBehavior + GlowDrawBehavior）
  └── ShaZhaoJiuYueZhan（+WaveBehavior + GlowDrawBehavior）

阶段 4：批次 3 — 特殊 AI 弹幕（每个 10-15 分钟）
  ├── StarDartProj（Gravity + Bounce + 命中随机转向）
  ├── StarRiverProj（StateMachineBehavior）
  └── PineNeedleProj（HomingBehavior）

阶段 5：剩余弹幕逐个确认并重构
```
