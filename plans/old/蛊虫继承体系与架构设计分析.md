## 一、蛊虫分类总览

蛊虫在 VerminLordMod 中分为三大类，每类有独立的基类和继承体系：

| 分类 | 基类 | 接口标记 | 数量 |
|------|------|---------|------|
| **武器蛊**（攻击蛊） | [`GuWeaponItem`](Content/Items/Weapons/GuWeaponItem.cs:26) | `IWeaponCanReforge`, `IGu` | ~50+ 种具体蛊虫 |
| **饰品蛊**（被动蛊） | [`GuAccessoryItem`](Content/Items/Accessories/GuAccessoryItem.cs:21) | `IAccCanReforge` | ~5+ 种 |
| **消耗蛊**（一次性使用） | [`GuConsumableItem`](Content/Items/Consumables/GuConsumableItem.cs:17) | 无 | ~10+ 种 |

## 二、蛊虫基类架构

### 2.1 武器蛊基类 — [`GuWeaponItem`](Content/Items/Weapons/GuWeaponItem.cs:26)

```
GuWeaponItem : ModItem, IWeaponCanReforge, IGu
```

**核心可重写字段（protected virtual）：**

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `needCtrl` | `true` | 是否需要炼化 |
| `qiCost` | `7` | 使用时消耗的真元 |
| `controlQiCost` | `10` | 每次炼化消耗的真元 |
| `unitConntrolRate` | `10` | 每次炼化增加的进度 |
| `uncontrolRate` | `0.01f` | 自然脱离速度（每 tick） |
| `_useTime` | `20` | 使用速度 |
| `_useStyle` | `Guitar` | 使用动画 |
| `_guLevel` | `1` | 蛊虫等级（一转=1） |
| `moddustType` | `-1` | 手持时虚影的粒子类型 |

**核心公共字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `hasBeenControlled` | `bool` | 是否已炼化完成 |
| `controlRate` | `float` | 当前炼化进度 [0, 100] |

**核心方法：**

- [`CanUseItem`](Content/Items/Weapons/GuWeaponItem.cs:101) — 右键炼化逻辑（消耗真元增加进度）；左键使用检查（炼化状态、等级惩罚、真元充足）
- [`UpdateInventory`](Content/Items/Weapons/GuWeaponItem.cs:146) — 每帧：进度达到100时标记已炼化；未炼化时按 `uncontrolRate` 衰减
- [`HoldItem`](Content/Items/Weapons/GuWeaponItem.cs:172) — 手持时生成虚影粒子
- [`SaveData/LoadData`](Content/Items/Weapons/GuWeaponItem.cs:186) — 持久化 `controlRate` 和 `hasBeenControlled`

### 2.2 饰品蛊基类 — [`GuAccessoryItem`](Content/Items/Accessories/GuAccessoryItem.cs:21)

```
GuAccessoryItem : ModItem, IAccCanReforge
```

与 `GuWeaponItem` 结构高度相似，但：

- 无 `Shoot` 方法（不发射弹幕）
- 有 [`CanEquipAccessory`](Content/Items/Accessories/GuAccessoryItem.cs:129) — 装备时检查真元消耗和境界防御上限
- 右键炼化逻辑与武器蛊一致

### 2.3 消耗蛊基类 — [`GuConsumableItem`](Content/Items/Consumables/GuConsumableItem.cs:17)

```
GuConsumableItem : ModItem
```

**抽象字段：**

| 字段 | 说明 |
|------|------|
| `QiCost` | 真元消耗量 |
| `GuLevel` | 蛊虫等级 |

**模板方法模式：**

- [`CanUseItem`](Content/Items/Consumables/GuConsumableItem.cs:69) — 检查真元 + 调用 `CanApplyEffect`（虚方法，子类可扩展）
- [`UseItem`](Content/Items/Consumables/GuConsumableItem.cs:87) — 强行调动惩罚 + 扣真元 + 调用 `ApplyEffect`（抽象方法，子类实现具体效果）

### 2.4 接口体系

| 接口 | 文件 | 说明 |
|------|------|------|
| [`IWeaponCanReforge`](Common/IWeaponCanReforge.cs:9) | 空接口 | 标记武器蛊可被重铸 |
| [`IAccCanReforge`](Common/IAccCanReforge.cs:9) | 空接口 | 标记饰品蛊可被重铸 |
| [`IGu`](Content/Items/IGu.cs:9) | 空接口 | 标记为蛊虫（预留扩展） |


- 每升一小阶段 +1 格

## 五、蛊虫战斗系统

### 5.1 蛊道媒介 — [`GuMediumWeapon`](Content/Items/Weapons/GuMediumWeapon.cs:21)

```
GuMediumWeapon : ModItem
```

这是玩家开窍后获得的**唯一武器**，自身无伤害，所有攻击来自空窍中启用的蛊虫。

**攻击流程：**

1. [`Shoot`](Content/Items/Weapons/GuMediumWeapon.cs:63) 被调用
2. 获取已启用的攻击蛊列表（`GetActiveAttackGus`）
3. 检查杀招配方（`MatchShaZhao`）
   - 匹配成功 → 释放杀招（冷却 5 秒，参与者休眠）
   - 匹配失败 → 平 A 齐射（`FireSalvo`）
4. 平 A 齐射：所有攻击蛊同时散射，每只蛊虫消耗 10% 占据额度

**杀招系统（MVA 硬编码）：**

当前仅有一条配方：
- **月光蛊 + 酒虫 = 酒月斩**（3 倍伤害，冷却 5 秒）

### 5.2 伤害类型

所有蛊虫使用统一的 [`InsectDamageClass`](Content/DamageClasses/InsectDamageClass.cs:6) 伤害类型。

### 5.3 弹幕类型获取

蛊虫的弹幕类型通过两种方式获取：
1. 优先使用 `Item.shoot`（每个蛊武器已正确设置）
2. 回退：通过 `ModItem` 类型匹配（兼容旧蛊虫）

## 六、蛊虫与外部系统的交互

### 6.1 真元系统

| 组件 | 文件 | 职责 |
|------|------|------|
| [`QiResourcePlayer`](Common/Players/QiResourcePlayer.cs:16) | 真元存储/消耗/恢复 | 提供 `QiAvailable`（可用真元 = 上限 - 占据额度） |
| [`QiRealmPlayer`](Common/Players/QiRealmPlayer.cs:16) | 境界计算 | 计算真元上限、恢复速率、空窍格子数 |
| [`QiTalentPlayer`](Common/Players/QiTalentPlayer.cs) | 资质加成 | 影响真元上限和恢复速率 |

**占据额度机制：**
- 每个蛊虫有 `QiOccupation`（= 10 + damage / 10）
- 启用蛊虫时，占据额度从可用真元中扣除
- 可用真元不足时无法启用新蛊虫

### 6.2 事件系统

| 事件 | 文件 | 触发时机 |
|------|------|---------|
| [`GuActivatedEvent`](Common/Events/GuWorldEvent.cs:46) | 蛊虫启用时 | 发布到 EventBus |
| [`PlayerGusLostOnDeathEvent`](Common/Events/GuWorldEvent.cs:54) | 玩家死亡时 | 包含叛逃/自毁/保留列表 |
| [`PlayerQiChangedEvent`](Common/Events/GuWorldEvent.cs:37) | 真元变化时 | 消耗/恢复/死亡清空 |
| [`PlayerRealmUpEvent`](Common/Events/GuWorldEvent.cs:192) | 境界提升时 | 大境界/小阶段突破 |

### 6.3 道痕冲突系统

[`DaoHenConflictSystem`](Common/Systems/DaoHenConflictSystem.cs:26) 预埋了道痕冲突检测框架：

- 16 种道痕路径（位掩码枚举）
- 冲突规则表（炎↔冰、光↔暗、血↔医）
- 冲突掩码计算接口
- 默认道痕映射表（MVA 阶段未填充具体蛊虫）

### 6.4 永久增益系统

[`GuPerkSystem`](Common/Players/GuPerkSystem.cs:24) 管理消耗类蛊虫的永久性增益：

| 增益类型 | 来源蛊虫 | 效果 |
|---------|---------|------|
| 白豕蛊之力 | 白豕蛊 | 近战伤害 +1%（上限一猪之力） |
| 黑豕蛊之力 | 黑豕蛊 | 近战伤害 +1%（可叠加白豕蛊） |
| 斤力蛊系列 | 斤力蛊/钧力蛊 | 近战伤害递增 |
| 酒虫系列 | 酒虫→四味→七香→九眼 | 真元恢复速度递增（+1→+2→+4→+8） |
| 额外寿命 | 十命蛊/百命蛊 | 生命上限加成 |
| 额外速度 | 龙珠蟋蟀 | 移速/加速度加成 |
| 一胎蛊 | 一胎蛊 | +1 召唤栏 |

### 6.5 合炼系统

[`GuCraftSystem`](Common/UI/KongQiaoUI/GuRecipe.cs:77) 管理所有配方注册和查询：

- 配方数据结构：[`GuRecipe`](Common/UI/KongQiaoUI/GuRecipe.cs:12)（产物、等级、真元消耗、材料、类别）
- 配方类别：力道蛊、豕蛊、酒虫、破境丹、沙砾、天牛、资质、消耗品
- 当前已注册配方：~20 条（一转至五转）
- UI：[`GuCraftUI`](Common/UI/KongQiaoUI/GuCraftUI.cs:20) — 现代化扁平轻量风格合炼面板

### 6.6 掉落系统

[`GlobalNPCLoot`](Common/GlobalNPCs/GlobalNPCLoot.cs) 管理所有 NPC 的蛊虫掉落，覆盖几乎所有原版 NPC 类型。

### 6.7 前缀系统

[`GlobalPrefix`](Common/GlobalItems/GlobalPrefix.cs) 通过 `IWeaponCanReforge` / `IAccCanReforge` 接口标记为蛊虫的物品自动分配蛊虫专属前缀。

## 八、架构设计评价

### 优点

1. **流派体系清晰**：50 个 Dao 抽象类通过极简的 `moddustType` 重写实现视觉差异化，扩展新蛊虫只需继承对应流派 + 重写基础属性
2. **炼化系统统一**：`GuWeaponItem` 和 `GuAccessoryItem` 共享相同的炼化流程（右键炼化 → 炼入空窍）
3. **空窍系统独立**：`KongQiaoPlayer` 从旧 `QiPlayer` 拆分，职责单一
4. **事件驱动**：蛊虫启用/死亡/真元变化通过 EventBus 解耦
5. **道痕预埋**：`DaoHenTags` 字段和冲突检测框架已预留，便于 P2 扩展

### 待改进

1. **空接口泛滥**：`IWeaponCanReforge`、`IAccCanReforge`、`IGu` 均为空接口，仅作为标记使用，应考虑合并或赋予实际方法
2. **流派抽象类冗余**：50 个流派类每个仅重写一个字段，可考虑用配置表或枚举替代
3. **杀招硬编码**：`GuMediumWeapon.MatchShaZhao` 中的杀招配方为硬编码，应抽离为可配置的配方系统
4. **弹幕类型回退**：`GetGuProjectileType` 中的类型匹配 switch 语句随蛊虫增多难以维护
5. **饰品蛊未接入空窍**：`GuAccessoryItem` 目前直接装备在饰品栏，未通过空窍系统管理
6. **道痕映射表为空**：`DefaultDaoHenMap` 在 MVA 阶段未填充任何具体蛊虫
