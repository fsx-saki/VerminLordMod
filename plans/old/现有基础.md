# VerminLordMod 模组架构分析报告

## 1. 项目概览

- **模组名称**: VerminLordMod（蛊师模组）
- **作者**: 風笙咲
- **版本**: 1.1.0.4
- **依赖**: SubworldLibrary@2.2.2.2
- **语言**: C# (tModLoader, .NET 8.0)
- **主题**: 基于《蛊真人》小说设定的修仙/蛊虫系统

## 3. 核心系统架构

### 3.1 玩家系统 ([`Common/Players/`](Common/Players/))

#### 重构后的五系统架构（L3 已完成）

##### [`QiResourcePlayer`](Common/Players/QiResourcePlayer.cs) - 真元资源系统（168行）✅
- **职责**: 真元存储、消耗、恢复、死亡清空
- **核心字段**: `QiMaxBase`, `QiMaxCurrent`, `QiCurrent`, `QiOccupied`, `QiAvailable`
- **恢复机制**: 离散恢复（计时器+每跳整点），保留旧公式
- **接口**: `ConsumeQi()`, `RefundQi()`, `OnDeathClearQi()`, `UpdateQiOccupied()`
- **事件**: 发布 `PlayerQiChangedEvent`
- **数据持久化**: 独立 `SaveData`/`LoadData`

##### [`QiRealmPlayer`](Common/Players/QiRealmPlayer.cs) - 境界系统（176行）✅
- **职责**: 境界存储、突破、破境提示、真元上限计算、空窍格子数计算
- **核心字段**: `GuLevel`(1-10), `LevelStage`(0-3), `BreakthroughProgress`
- **方法**: `OnAwakening()`, `StageUp()`, `LevelUp()`, `ApplyRealmEffects()`
- **公式**: 保留旧 `SetQis` 公式用于真元上限和恢复速率
- **新维度**: 空窍格子数公式 `BASE_KONGQIAO_SLOTS + (GuLevel-1)*2 + LevelStage`
- **事件**: 发布 `PlayerRealmUpEvent`

##### [`QiTalentPlayer`](Common/Players/QiTalentPlayer.cs) - 资质系统（105行）✅
- **职责**: 资质存储、旧值映射、修炼效率加成
- **枚举**: `TalentGrade` (Ding/Bing/Yi/Jia)
- **旧值映射**: `GetZiZhiMultiplier()` 返回与旧 `(int)ZiZhi` 等价的数值
- **方法**: `OnAwakening()`, `CalculateEffects()`, `FromLegacyZiZhi()`

##### [`KongQiaoPlayer`](Common/Players/KongQiaoPlayer.cs) - 空窍系统（340行）✅
- **职责**: 蛊虫炼化/取出/启用/休眠、容量管理、死亡处理
- **核心类**: `KongQiaoSlot`（含忠诚度、炼化度、道痕标签等字段）
- **方法**: `TryRefineGu()`, `TryExtractGu()`, `SetGuActive()`, `OnPlayerDeath()`
- **占据额度**: 启用蛊虫消耗真元额度，死亡时全部休眠
- **忠诚度**: 死亡时忠诚度 < 40% 的蛊虫可能叛逃或自毁
- **事件**: 发布 `GuActivatedEvent`, `PlayerGusLostOnDeathEvent`

##### [`GuPerkSystem`](Common/Players/GuPerkSystem.cs) - 永久增益系统（318行）✅
- **力量类**: 白豕/黑豕/斤力/钧力蛊（含上限控制）
- **寿命类**: 十命/百命/千命蛊
- **速度类**: 龙珠蟋蟀
- **召唤类**: 一胎蛊
- **酒虫系列**: 酒虫→四味→七香→九眼
- **已迁移的战斗数值修改**: `ModifyWeaponDamage`, `ModifyWeaponKnockback`, `ModifyMaxStats`, `PostUpdateRunSpeeds`, `PostUpdateEquips`

##### [`QiPlayer`](Common/Players/QiPlayer.cs) - 旧核心系统（432行，待删除 ⏳）
- **当前状态**: 保留中，包含 `[Obsolete]` 兼容层和未迁移的 Buff 反击逻辑
- **剩余未迁移**: `OnHitByNPC`（火衣/水壳/映伤反击）、`CanUseItem`（木魅攻击替代）、`AddStartingItems`、`showInfo`
- **删除计划**: 待 Buff 反击逻辑迁移到专用 `GlobalPlayer` 后删除
- **⚠️ Bug**: [`PostUpdateEquips`](Common/Players/QiPlayer.cs:247) 中存在重复的 `LittleSoulbuff` 检查（第251行和第255行），同一buff分别 +1 和 +2 召唤栏，疑似应为 `BigSoulbuff` 的笔误

#### [`GuWorldPlayer`](Common/Players/GuWorldPlayer.cs) - 蛊世界模拟状态（395行）
- **家族声望系统**: `Dictionary<FactionID, FactionRelation>` 存储每个势力的声望点数
- **通缉系统**: `List<BountyStatus>` 活跃通缉列表
- **背刺记录**: `List<BetrayalRecord>` 记录玩家的背刺历史
- **恶名/声望**: `InfamyPoints`（影响所有家族初始态度）、`FamePoints`（影响中立NPC态度）
- **信念辅助**: 提供从玩家侧查询NPC信念状态的辅助方法
- **数据持久化**: 通过 `SaveData`/`LoadData` 保存

#### [`ChunQiuChanPlayer`](Common/Players/ChunQiuChanPlayer.cs) - 春秋蝉回溯系统（271行）
- **即死回溯**: 玩家受到即死伤害时自动触发，防止死亡
- **回溯动画**: 旋转+滑行回记录点
- **保护机制**: 跨帧持续保护（330 ticks = 5.5秒），强制保持存活，保护真元不被减半
- **时光能量**: 与 [`ChunQiuChan`](Content/Items/Weapons/ChunQiuChan.cs) 物品联动

#### [`EffectsPlayer`](Common/Players/EffectsPlayer.cs) - 特效系统（110行）
- 处理玩家身上的Buff视觉效果
- 当前实现了旋风特效（`BreezeWheelbuff`），含拖尾渲染
- 通过 [`PlayerEffectDrawSystem`](Common/Systems/PlayerEffectDrawSystem.cs) 在 `PostDrawInterface` 中调用绘制
- 使用 Additive 混合模式绘制旋风效果

#### [`ProjectileTrailPlayer`](Common/Players/ProjectileTrailPlayer.cs) - 投射物拖尾

  - **信念系统**: `BeliefState` 结构体（黑暗森林核心），包含风险阈值 `RiskThreshold`、置信度 `ConfidenceLevel`、实力估计 `EstimatedPower`、交互历史等

- [`GuMasterBase`](Content/NPCs/GuMasters/GuMasterBase.cs) - 蛊师NPC抽象基类（836行）
  - 整合 `IGuMasterAI` 智能接口
  - **信念黑箱**: `Dictionary<string, BeliefState> PlayerBeliefs` 替代确定性态度计算
  - **弹幕保护系统**: `ProjectileProtectionEnabled` 防止玩家弹幕误伤
  - **声望联动**: 与 `GuWorldPlayer` 声望系统交互
  - **修为系统**: 与 `QiPlayer` 真元关联
  - **对话/战斗双模式**: 态度友好时可对话，敌对时战斗
  - **目击者系统**: `HasWitnesses()` 检测附近是否有目击者，击杀玩家影响声望
  - **盟友警报**: `AlertNearbyAllies()` 通知附近友军NPC
  - **子类需重写**: `SetupGuMaster()`, `GetPersonality()`, `GetFaction()`, `GetRank()`

- [`GuYuePatrolGuMaster`](Content/NPCs/GuMasters/GuYuePatrolGuMaster.cs) - 古月巡逻蛊师（647行）
  - **掠夺型原型**: 面对弱者动手，面对强者回避，面对未知试探
  - **信念驱动**: 基于 `RiskThreshold` 决定观望/试探/动手/回避，`RiskThreshold` 上限0.5（始终激进）
  - **搜尸逻辑**: 击败玩家后从背包取走元石和未炼化蛊虫
  - **记忆机制**: 被击败后记住玩家，下次遇到更谨慎
  - **巡逻AI**: 在出生点周围巡逻，检测玩家进入感知范围
  - **战斗AI**: 远程月光弹幕，含弹道预判（`AimAhead()`）
  - **呼叫支援**: `CallForHelp()` 召唤2-3名增援
  - **生成条件**: 仅在 `GuYueCompoundBiome` 中刷新

  - **家族/势力枚举**: `FactionID`（古月、白家、熊家、铁家、百家、汪家、赵家、贾家、散修）
  - **声望等级**: `RepLevel`（敌对/不友好/中立/友好/盟友）
  - **交互类型**: `InteractionType`（对话/交易/攻击/挑衅/结盟/背刺/悬赏/威吓/贿赂）
  - **世界事件类型**: `WorldEventType`（家族战争/蛊师猎杀/宝藏现世/家族集会/商队到来）
  - **蛊师修为等级**: `GuRank`（一转初阶~九转巅峰）
  - **蛊师性格**: `GuPersonality`（谨慎/激进/贪婪/友善/狡猾/鲁莽）
  - **蛊师AI状态**: `GuMasterAIState`（空闲/巡逻/观察/试探/战斗/逃跑/呼叫支援）
  - **蛊师态度**: `GuAttitude`（忽略/观望/警惕/敌对/恐惧）
  - **家族关系矩阵**: `Dictionary<FactionID, FactionState>` 存储家族间好感度，基于小说设定初始化默认关系
  - **数据持久化**: 通过 `SaveWorldData`/`LoadWorldData` 保存家族关系变化
  - **⚠️ 代码债务**: 包含未使用的 `ActiveEvents` 字段（实际事件调度在 WorldEventSystem 中），L1 决策 D-01 计划与 WorldEventSystem 合并为 WorldStateMachine
  - **工具方法**: 家族名称显示、声望等级颜色等
- [`WorldEventSystem`](Common/Systems/WorldEventSystem.cs) - 世界事件触发系统（305行）
  - **事件队列**: `ActiveEvents` 和 `EventHistory`
  - **周期性事件**: 商队（每7天）、兽潮（每15天）、家族集会（每30天）
  - **事件实例**: `WorldEventInstance` 包含类型、持续时间、位置、名称
  - **与 GuWorldSystem 联动**: 事件触发时影响家族声望和NPC行为
- [`DaoHenConflictSystem`](Common/Systems/DaoHenConflictSystem.cs) - 道痕冲突系统（311行）🆕
  - **道痕枚举**: `DaoPath` flags 枚举（16种道痕路径）
  - **冲突掩码**: `ConflictMask` 结构体，支持 `SetConflict()`/`HasConflict()`/`Merge()`
  - **冲突规则**: `ConflictRules` 字典定义道痕间的冲突关系
  - **⚠️ MVA 阶段**: `DefaultDaoHenMap` 为空字典，尚未注册任何实际的道痕标签映射

## 4. 数据流与关键交互

### 4.1 真元消耗流程
```
玩家使用武器蛊虫
  → GuWeaponItem.CanUseItem() 检查 qiCurrent >= qiCost
  → GuWeaponItem.UseItem() 扣除 qiCost
  → QiResourcePlayer.PostUpdate() 每帧恢复真元
```

### 4.2 炼化流程
```
右键使用蛊虫（altFunctionUse == 2）
  → 检查 qiCurrent >= controlQiCost
  → 扣除真元，增加 controlRate
  → UpdateInventory() 中 controlRate 缓慢下降（uncontrolRate）
  → controlRate 达到100时 hasBeenControlled = true
  → 后续右键自动触发 KongQiaoPlayer.TryRefineGu() 炼化入窍
```

### 4.3 境界突破流程
```
使用舍利蛊 或 积累 levelStageUpRate
  → QiRealmPlayer.StageUp() 提升小阶段
  → QiRealmPlayer.LevelUp() 提升大转数
  → ApplyRealmEffects() 重新计算真元上限和恢复速度
```

### 4.4 狼潮事件流程
```
每天清晨概率触发
  → WolfSystem.isWolfWave = true
  → WolfWaveRate 逐渐增加
  → 达到80%时可能召唤雷冠头狼
  → 达到100%时狼潮结束
```

### 4.5 蛊师NPC信念驱动流程
```
每帧更新
  → GuMasterBase.AI() 主循环
    → Perceive() 感知环境（目标距离、生命、真元、附近友军/敌军）
    → UpdateBelief() 更新对目标玩家的信念状态
      → 观察玩家行为（生命、装备、战斗表现）
      → 动态调整 RiskThreshold 和 EstimatedPower
      → 增加 ConfidenceLevel
    → CalculateAttitude() 基于信念计算态度
    → Decide() 基于信念做出决策
      → RiskThreshold < 0.3 → 主动攻击
      → RiskThreshold 0.3-0.5 → 试探
      → RiskThreshold 0.5-0.7 → 观望
      → RiskThreshold 0.7-0.9 → 回避
      → RiskThreshold > 0.9 → 逃跑
    → ExecuteAI() 执行对应行为
```

### 4.6 声望交互流程
```
玩家与蛊师NPC交互
  → GuMasterBase.HandleInteraction()
    → 更新 GuWorldPlayer.FactionRelations
    → 根据交互类型增减声望点数
    → 检查是否触发通缉/悬赏
    → 更新 NPC 的 BeliefState
  → 声望 UI 自动刷新显示
```

### 4.7 世界事件触发流程
```
每天黎明
  → WorldEventSystem.PreUpdateWorld()
    → 更新事件计时器
    → 检查各事件触发条件
    → 创建 WorldEventInstance 加入 ActiveEvents
    → 事件激活时影响 NPC 行为和世界状态
```

### 4.8 春秋蝉回溯流程
```
玩家受到即死伤害
  → ChunQiuChanPlayer.PreKill() 拦截
    → 检查是否持有炼化后的春秋蝉
    → 检查时光能量 > 0
    → 触发回溯：旋转滑行回记录点
    → 消耗全部能量，恢复对应百分比生命和真元
    → 进入跨帧保护状态（330 ticks）
  → ChunQiuChan 物品同步消耗能量
```
