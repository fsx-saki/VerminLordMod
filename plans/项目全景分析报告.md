## 一、项目概览

| 项目 | 值 |
|------|-----|
| **Mod 名称** | VerminLordMod（蛊真人Mod） |
| **版本** | 1.1.0.4 |
| **作者** | 風笙咲 |
| **平台** | Terraria tModLoader (1.4.4, .NET 8.0) |
| **外部依赖** | SubworldLibrary, ParticleLibrary |
| **核心主题** | 蛊师世界模拟 — 黑暗森林、信息不透明、不可逆性、零和博弈、维度隔离 |
| **代码总量** | ~55,000+ 行 C#（含 ~130 个弹幕文件、~60 个 Buff 文件、~100 个武器文件、~20 个 NPC 文件、~30 个 UI 文件） |
| **设计哲学** | ① 黑暗森林（任意角色可能背叛）② 信息不透明（NPC 信念为黑盒）③ 不可逆性（永久性世界改变）④ 零和博弈（资源稀缺）⑤ 维度隔离（空窍 vs 背包） |

### 3.6 Zero 基础蛊虫体系（★ v3.0 新增 — 项目最大新模块）

这是项目最新、架构最完整的模块。为 50 种道 + 阴阳道（共 51 种）各设计一种基础蛊虫。

**三层接口分离架构**（完全脱耦合）：
```
DaoWeapon (蛊虫物品)
├── IKinematicProvider      → 弹道参数（速度/数量/散布）
├── IOnHitEffectProvider    → 命中效果（标签/持续时间/数值）
└── ITacticalTriggerProvider → 战术触发（条件触发增益）

BaseBullet (弹射物, sealed 生命周期)
└── List<IBulletBehavior>   → 行为组合（43 种可用行为组件）
```

**文件结构**：
- `Content/Items/Weapons/Zero/` — 51 个 BaseGu 物品（FireBaseGu.cs 等）
- `Content/Projectiles/Zero/` — 51+ 个 BaseProj + 变体弹幕
  - Fire: 10 个变体（Bomb/Charge/Explosion/FlameWall/Meteor/Rain/Rise/Scatter/Stream）
  - Water: 11 个变体（Arc/Drop/Formation/FormationBolt/Heal/RingConverge/Slash/Swing/Vortex/Wave）
  - Ice: 5 个变体（Blizzard/Explosion/Field/ShardScatter）
  - Wind: 5 个变体（Burst/Cyclone/Storm/Wall）
  - Lightning: 5 个变体（Bolt/Chain/EMP/Storm）
  - Dark: 4 个变体（Nova/Tentacle/Vortex）
- **状态**：所有 51 个蛊虫物品 + 51+ 个弹射物已生成，C# 编译通过，脱耦合架构已验证成功
- **差异化程度**（v3.1 修正 — v3.0 称"全部相同"是错误的）：
  - **BaseGu 物品层**：6 种核心道（Fire/Water/IceSnow/Wind/Lightning/Dark）已实现多模式攻击切换系统（5-6 种模式，各有不同弹幕类型/速度/伤害倍率/使用间隔）；其余 45 种通过 IOnHitEffectProvider 实现了差异化的命中效果标签（Silence/DoT+LifeSteal/Blind/ArmorShred/Slow+Pull/DoT+Fear 等）
  - **BaseProj 弹幕层**：~18 种已实现差异化行为组合（Fire: Gravity+Bounce+LiquidTrail+ExplosionKill；Water: Gravity+ParticleBody+WaterTrail+LiquidBurst+SuppressDraw；Dark: Homing+DustTrail+GlowDraw+Splash；Wind: Aim+Wave+DustTrail+GlowDraw+Splash；Lightning: Aim+DustTrail+GlowDraw+Splash+连锁闪电；Light: Aim+DustTrail+GlowDraw+Splash(Forward)；Blood: Aim+DustTrail+GlowDraw+Splash(Radial)；IceSnow: Aim+DustTrail+DustKill 等）；~33 种为简单 AimBehavior 占位
- **待完成**：~33 种简单 BaseProj 行为差异化、占位贴图替换

### 9.3 「感觉凌乱」的根本原因分析

你的项目**并不缺代码**——恰恰相反，代码量已经很大且架构质量不错。凌乱感来自：

1. **文档严重滞后于代码**：v2.0 报告将大量已实现的系统标为「缺失」或「待实施」，导致你误以为还有很多基础工作没做
2. **Zero 体系进度被低估**：v3.0 称"51 种蛊虫行为相同"是错误的——实际上 6 种核心道已实现多模式攻击系统，~18 种 BaseProj 已差异化。真正待完成的是 ~33 种简单 BaseProj 的行为设计。这比"全部从零开始"的工作量小很多
3. **P0 缺陷未修复**：元石掉落和 DefaultDaoHenMap 是两个一行代码就能修但一直没修的问题，给人「项目不完整」的感觉
4. **缺乏可玩的 gameplay 循环**：系统很多但串联不够——觉醒→修炼→战斗→交易→声望的完整循环还没跑通
5. **贴图大量缺失**：视觉反馈不足加剧了「不完整」的感觉
