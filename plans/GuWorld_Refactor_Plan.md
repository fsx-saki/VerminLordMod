# 蛊师世界重构计划：从 RPG 声望系统到黑暗森林 MVA

> 基于四次迭代的最终定稿
> 核心原则：**你永远不知道眼前这个巡逻蛊师，是因为你弱而动手，还是因为你强而回避，还是在试探你的底牌**

---

## 一、演进脉络

| 版本 | 核心问题 | 关键修正 |
|------|---------|---------|
| **v0 原始规划** | 传统 RPG 声望任务系统，40+文件铺开 | 识别出「换皮 RPG」风险 |
| **v1 重构建议** | 提出效用计算、双层状态、道痕系统，但过度工程化 | 引入「利益-风险」逻辑，但公式过于精确 |
| **v2 修正方案** | 转向信念分布、行为一致性、情境策略，但保留安全区、瀑布式开发 | 不确定博弈方向正确，但实现路径复杂 |
| **v3 MVA 草案** | 收缩为单文件验证，但固定原型导致可预测，且未解决 TR 物品系统冲突 | 最小化思路正确，但忽略空窍-背包底层矛盾 |
| **最终定稿** | 确定空窍双轨制、真元预算、媒介触发、无安全区、个体级信念 | 在 TR 框架内重建蛊界核心体验 |

## 二、最终架构核心机制

### 2.1 空窍双轨制（解决 TR 物品系统冲突）

```
┌─────────────────────────────────────────┐
│  空窍（ModPlayer 数据结构）              │  ← 已炼化蛊虫
│  - 独立 UI（K 键或物品触发）              │
│  - NPC 无法直接搜刮                       │
│  - 死亡时按忠诚度/自毁机制处理            │
│  - 容量上限以「真元占据」软约束           │
├─────────────────────────────────────────┤
│  背包/箱子（TR 原生）                    │  ← 未炼化蛊虫、材料、元石
│  - NPC 搜尸/搜基地时掠夺对象              │
│  - 可交易、丢弃、存放                     │
└─────────────────────────────────────────┘
```

**关键规则**：
- 已炼化蛊虫进入空窍，不占用背包格子，NPC **无法搜刮**
- 未炼化蛊虫、元石、材料留在物理世界，**可被掠夺**
- 死亡时：空窍中本命蛊保留（或按忠诚度叛逃/自毁），背包物品被 NPC 搜走

### 2.2 真元预算制（替代硬性格子限制）

| 机制 | 说明 |
|------|------|
| **真元上限** | 随修为提升（一转 100，二转 200...） |
| **占据额度** | 启用蛊虫时一次性扣除，停用后归还。玉皮蛊占 20，月光蛊占 30 |
| **运行消耗** | 催动攻击时额外扣真元。平 A 每只扣 2-5 点 |
| **软约束** | 玩家可以塞 10 只蛊虫进空窍，但启用 4 只就占满真元，无法启用第 5 只 |

**体验目标**：不是系统说「你不能带」，而是玩家自己发现「塞满攻击蛊后没真元开防御，被散修捡尸」。

### 2.3 蛊虫状态与交互

| 状态 | 效果 | 切换方式 |
|------|------|---------|
| **休眠** | 不消耗真元，不暴露气息，不生效 | 空窍 UI 点击 |
| **启用** | 占据真元额度，被动蛊自动生效，攻击蛊等待触发 | 空窍 UI 点击 |
| **催动（攻击时）** | 额外消耗真元，释放弹幕 | 媒介武器攻击/快捷键 |

**本命蛊**：可更换，优先保留，维持代价更低（MVA 阶段可暂不区分）。

### 2.4 战斗系统：媒介触发 + 杀招配方

**媒介武器**：一把「蛊道媒介」（如改造后的 Moonlight），自身无伤害，纯触发器。

**攻击流程**：
```
玩家按攻击键 → 媒介 Shoot() → 查询空窍中启用的攻击蛊
  → 检查是否匹配杀招配方
    → 匹配：释放杀招弹幕 → 参与者集体休眠 → 进入冷却
    → 不匹配：所有启用的攻击蛊齐射弹幕 → 各自扣真元
```

**杀招配方（MVA 阶段硬编码 1 条）**：
- 月光蛊 + 酒虫 → 「酒月斩」（高伤弹幕，释放后两者休眠，冷却 5 秒）

**被动蛊**（如玉皮蛊）：在 `ModPlayer.PostUpdate()` 中每帧自动生效，不依赖攻击键。

### 2.5 NPC AI：信念黑箱 + 原型 + 无安全区

**信念分布（NPC 内部黑箱）**：
- NPC 对玩家修为/底牌持概率分布，而非点估计
- 置信度低时表现为观望/试探，置信度高时果断行动
- 玩家通过 NPC 行为反推，**无系统提示暴露数值**

**三种原型**：
| 原型 | 核心参数 | 行为特征 |
|------|---------|---------|
| **掠夺型** | `RiskThreshold` 动态漂移 | 面对弱者评估后动手，面对强者回避，面对未知试探 |
| **交易型** | `CooperationBias` | 优先交易，坐地起价，必要时结盟 |
| **隐忍型** | `PatienceFactor` | 长期观察，收集情报，伺机背刺 |

**RiskThreshold 动态漂移**（关键设计）：
- 初始 0.9（极度谨慎）
- 观察到玩家弱 → 降至 0.3（激进）
- 观察到玩家强/被反杀 → 升至 0.95（回避）
- **同一 NPC，同一玩家，不同阶段，行为完全不同**

**无安全区 + 明面/暗面规则**：
- 家族核心区：NPC **不能公开战斗**，但可以下毒、嫁祸、诱导、设局
- 家族边缘/野外：NPC 可公开抢夺、击杀
- 基地不主动被拆：NPC 只攻击**在场且携带高价值物品**的玩家，不拆空建筑
- 玩家可通过防御工事（迷踪阵、守卫）提高 NPC 的 `RiskThreshold`

**个体级信念**：每个 NPC 对每个玩家维护独立信念分布（以 `Player.name` 为 key），多人模式下玩家 A 被某 NPC 骗过，不影响玩家 B 面对该 NPC 的体验。

### 2.6 死亡惩罚（替代寿元机制）

| 惩罚 | 效果 |
|------|------|
| **真元清空** | 所有启用蛊虫失效，进入虚弱期，需打坐恢复 |
| **背包搜刮** | 元石、未炼化蛊虫、材料被击杀者（NPC/玩家）拿走 |
| **虚弱期** | 真元归零期间无法催动蛊虫，只能用原版武器 |

**复仇循环**：NPC 拿走你的元石后，你可以反杀该 NPC 夺回。

### 2.7 开局方式：天外道痕

**流程**：
1. **Day 1**：玩家是凡人，无空窍，无真元，用原版装备
2. **开窍事件**：击杀第一只「野蛊」（替换原版特定怪物，如丛林黄蜂 → 蜂刺蛊）
3. **获得空窍石**：使用后开启空窍，一转初阶，容量 3 格（真元上限 100）
4. **本命蛊**：第一只炼化的蛊虫自动成为本命蛊

**与 TR 原生融合**：前期依赖原版装备，开窍后逐步替换为蛊虫能力。

### 2.8 蛊虫获取：嵌入 TR 生态位

| 来源 | 机制 | 稀缺性 |
|------|------|--------|
| **野生蛊虫** | 替换原版特定怪物，击杀掉落未炼化蛊虫 | 掉落率 2%-5%，每世界储量有限 |
| **炼制** | 蛊盅 + 材料 + 元石 + 特定环境，有失败率 | 可控但风险高 |
| **交易** | 城镇 NPC 出售，价格随玩家急需程度浮动 | 零和博弈 |
| **掠夺** | 击杀 NPC/玩家，搜刮其背包中的未炼化蛊虫 | 无法掠夺本命蛊 |

### 2.9 玩家如何查看空窍

**MVA 阶段最简 UI**：
- 右键点击「空窍石」物品打开面板
- 10 格面板，显示已炼化蛊虫贴图
- 悬停：名称 + 占据真元 + 炼化度
- 右键格子：取出到背包 / 启用/休眠切换

## 三、实施计划（MVA 阶段）

### 目标
只验证一个核心问题：**玩家是否无法预测古月巡逻蛊师下一秒的行为？**

### 修改文件（4 个）

| 文件 | 修改内容 |
|------|---------|
| [`GuYuePatrolGuMaster.cs`](Content/NPCs/GuMasters/GuYuePatrolGuMaster.cs) | **超级原型**：信念黑箱 + 掠夺型动态漂移 + 无安全区 + 搜尸 |
| [`GuMasterBase.cs`](Content/NPCs/GuMasters/GuMasterBase.cs) | 新增 `Perceive()` 感知接口、`OnLoot()` 搜尸接口 |
| [`GuWorldPlayer.cs`](Common/Players/GuWorldPlayer.cs) | 新增 `KongQiao` 空窍数据结构、真元系统、个体级信念存储 |
| [`Content/Items/Weapons/GuMedium.cs`](Content/Items/Weapons/GuMedium.cs) | 媒介武器（空壳触发器，查询空窍发射弹幕） |

### 验证场景（5 个）

| 场景 | 预期行为 | 验收标准 |
|------|---------|---------|
| 新手首次遇到 | RiskThreshold 0.9 → 观望/保持距离 | 玩家感受到「他在观察我，但没动手」 |
| 玩家展示酒虫 | 信念更新 → 试探（询问/接近） | 玩家感受到「他注意到酒虫了」 |
| 玩家暴露弱点 | RiskThreshold 降至 0.3 → 突然动手 | 玩家感受到「他是因为我弱才动手的」 |
| 玩家反杀后 | RiskThreshold 升至 0.95 → 回避 | 玩家感受到「他记得我，他怕了」 |
| 玩家在家族核心区 | 不能公开战斗 → NPC 选择下毒/嫁祸 | 玩家感受到「表面安全，但可能被阴」 |

### 时间线（2 周）

| 天数 | 内容 |
|------|------|
| Day 1-2 | 空窍数据结构 + 真元系统 + 最简 UI |
| Day 3-4 | 巡逻蛊师信念黑箱 + 掠夺型原型 + 动态 RiskThreshold |
| Day 5 | 媒介武器 + 杀招配方（硬编码 1 条）+ 齐射逻辑 |
| Day 6-7 | 无安全区 + 明面/暗面规则 + 搜尸系统 |
| Day 8-10 | 调试参数（信念更新速率、试探频率、漂移阈值） |
| Day 11-14 | 验证 5 个场景 → 通过则抽象为基类，失败则调参 |

## 四、关键设计决策与理由

| 决策 | 放弃的方案 | 理由 |
|------|-----------|------|
| **空窍双轨制** | 纯 TR 背包 / 纯空窍 | 既保留 TR 建造根基，又实现本命蛊不可夺 |
| **真元预算制** | 硬性格子上限 | 软约束更有博弈感，玩家自己承担塞满的后果 |
| **无安全区** | 出生点 200 格安全区 | 安全区消解黑暗森林紧张感；空窍已提供维度安全 |
| **个体级信念** | 势力级信念 | 信息不透明是核心体验，不能多人共享 |
| **媒介武器触发** | 蛊虫直接使用 | 空窍中的蛊虫不是 Item，无法走原版 useStyle |
| **杀招配方硬编码起步** | 通用配方系统 | MVA 阶段只验证一个杀招循环，通过后再抽象 |
| **去掉寿元** | 死亡折寿永久惩罚 | 保留经济+虚弱惩罚已足够，避免过度压抑 |
| **基地不主动被拆** | NPC 可拆空建筑 | 保留 TR 建造乐趣，只攻击在场的玩家 |

## 五、后续扩展路线（MVA 验证通过后）

```
Phase 2：抽象基类
  → GuMasterBase 纳入信念系统 + 原型参数
  → 新增 3-4 个古月山寨 NPC（药姬、青书、伪装商人、落魄散修）

Phase 3：蛊虫生态
  → 5 种核心道痕（炎、冰、力、风、血）
  → 蛊虫维持代价（炼化度衰减、喂养）
  → 10-15 条杀招配方

Phase 4：家族扩展
  → 白家/熊家/铁家巡逻蛊师
  → 家族关系网络（古月+30白家，-20熊家）

Phase 5：小世界
  → SubworldLibrary 家族领地
  → 福地系统（玩家私人空间）

Phase 6：天道与情报
  → 天劫系统（可规避）
  → 情报网络（NPC 主动收集/放出假情报）
  → 执念系统（核心硬约束）
```

---

**最终判断**：这份计划从 40+ 文件的 RPG 声望系统，收敛到 **4 个文件、1 个 NPC、5 个验证场景**。核心体验不再是「完成任务刷声望」，而是 **「你永远不知道眼前这个巡逻蛊师，是因为你弱而动手，还是因为你强而回避，还是在试探你的底牌」**。
